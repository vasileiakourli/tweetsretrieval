Η διαδικασία της συλλογής των tweets ξεκινά με το να φτιάξω ένα application στο twitter.com . 
Πήρα τα κλειδιά μου και έφτιαξα ένα java project Information Retrieval. Έχοντας κατεβάσει την 
βιβλιοθήκη twitter4j-core.jar, την έκανα add external jar, ώστε να μπορέσω να χρησιμοποιήσω μεθόδους 
της. Μέσα σε αυτό το project  έφτιαξα 2 κλάσεις. Στην RestAPI.java έκανα την συλλογή των tweets και την 
τοποθέτησή τους στη βάση μου και στην Results.java ζητάω από τη βάση μου τα αποτελέσματα που θέλω για κάθε κόμμα.
Στην RestAPI.java xρησιμοποίησα τα κλειδιά που πήρα όταν έφτιαξα το application, ώστε 
να πραγματοποιήσω σύνδεση με το twitter. Επίσης, εδώ φτιάχνω τη βάση, εκτός αν υπάρχει ήδη. 
Έπειτα, φτιάχνω τον πίνακα που θα συγκεντρώσω τα tweets, πάλι σε περίπτωση που δεν τον έχω ήδη 
φτιάξει. Καλώ την μέθοδο getFromQuery, ώστε να μαζέψω σε μια λίστα τα tweets με τα συγκεκριμένα 
hashtags ή το @username του Τσίπρα και για όλες τις λίστες καλώ ξεχωριστά την insertInDataBase 
για να βάλω τα tweets στη mysql βάση δεδομένων. Ύστερα κλείνω τη σύνδεση μου με τη βάση δεδομένων.

Αναλυτικότερα, σε αυτήν την κλάση έχω τις εξής μεθόδους με τις αντίστοιχες λειτουργίες τους:
-Η μέθοδος getFromQuery, αυτό που κάνει είναι να ψάχνει το συγκεκριμένο hashtag που του δίνουμε και 
να το βάζει σε μία λίστα η οποία επιστρέφεται κιόλας. Αν συνυπάρχουν τα 2 κόμματα σε ένα φροντίζω να μην μαζεύω αυτό το tweet. 
-Στην insertInDataBase τώρα, βάζω τα tweets από τη λίστα, που δίνω ως όρισμα, στη βάση μου. Παίρνω 
το id του user, το id του tweet, το name του χρήστη, το text του tweet, αναλόγως αν υπάρχει βάζω και μια στήλη με το emoticon 
και τέλος περνάω και το κόμμα για το οποίο αναφέρεται το post σε μία στήλη. Το κόμμα το βρίσκω ελέγχοντας τι hashtag εχει 
στο κείμενο το tweet. Αν είναι tweet που μαζεύουμε το όνομα του πολιτικού αρχηγού βάζουμε "-" 
στο κόμμα. Εναλλακτικά, θα μπορούσα στα post που βρίσκω @atsipras να του βάλω σύριζα σαν κόμμα, 
αλλά προτίμησα να μετρήσω μόνο τα tweets που έχουν hashtag. Τέλος, τα βάζω στη βάση δεδομένων.
Η κλάση Results.java κάνει σύνδεση με τη βάση και πραγματοποιεί queries, που ζητάνε τα θετικά 
tweets και τα αρνητικά tweets για κάθε κόμμα, ενώ στο τέλος φροντίζω ώστε να εκτυπώνονται.  

Όλα αυτά τα μάζεψα τρέχοντας το RestAPI κάποιες φορές την μέρα.
Το παραδοτέο τώρα περιέχει ένα create_load table.sql μέσα στο οποίο δημιουργώ τον πίνακα που 
θα φορτώσουμε τα δεδομένα της μιας εβδομάδας. Μετά την εκτέλεση αυτού μπορούμε να εκτελέσουμε 
την Results.java για να πάρουμε τα παραπάνω αποτελέσματα.
